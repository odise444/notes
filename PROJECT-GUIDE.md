# 블로그 운영 이야기

## 이게 뭔가

mcu2cloud.kr이라는 블로그를 운영하고 있다. 펌웨어 개발하면서 삽질한 내용, 만든 도구들, 배운 것들 정리해두는 곳이다.

Hugo라는 정적 사이트 생성기 쓰고 있고, GitHub Pages로 무료 호스팅 받는다. main 브랜치에 push하면 자동으로 빌드되어서 올라간다. 편하다.

저장소는 https://github.com/odise444/notes 여기다.

## 어떻게 구성되어 있나

작업은 `D:\WorkSpace\notes\`에서 한다.

글들은 `content/posts/` 밑에 시리즈별로 폴더 나눠서 관리한다. BMS 개발기, Ghidra 역분석 이런 식으로. 한 폴더에 다 때려넣으면 나중에 찾기 힘드니까.

웹앱도 몇 개 만들어서 올려뒀는데, `static/` 폴더에 넣어두면 Hugo가 빌드할 때 그대로 복사해준다. 평면도 그리는 거, 수학 문제 푸는 거, PDF를 마크다운으로 바꾸는 거 세 개 있다.

## 지금까지 뭘 썼나

150개 정도 썼다. 돌아보니까 꽤 됐네.

BMS 개발기가 36개로 제일 많다. 배터리 관리 시스템 만들면서 진짜 많이 삽질했다. 그 다음이 Ghidra로 펌웨어 뜯어본 이야기 39개. Floor Planner 웹앱 만든 거 20개, STM32 부트로더 직접 짠 이야기 20개, 수학 문제 앱 15개, PlatformIO 세팅 가이드 15개. 최근에 PDF2MD 개발기 5개 추가했다.

다 각자 폴더에 정리되어 있어서 관리하기 편하다.

## 글 쓸 때 신경 쓰는 것들

파일 맨 위에 이런 거 넣어야 Hugo가 알아먹는다:

```yaml
---
title: "시리즈명 #번호 - 제목"
date: 2026-01-21
tags: ["태그1", "태그2"]
categories: ["카테고리"]
series: ["시리즈명"]
summary: "한 줄 요약"
---
```

글 스타일은 그냥 평소 말하듯이 쓴다. 딱딱하게 "~합니다" 이러면 쓰기도 힘들고 읽기도 지루하다. 코드는 실제로 돌아가는 거 위주로 넣고, 길어지면 적당히 끊는다. 한 번에 너무 많이 쓰면 나도 지치고 읽는 사람도 지친다.

파일명은 `bms-01.md`, `ghidra-15.md` 이런 식으로 시리즈 약어랑 번호 붙인다.

## 웹앱 올리는 법

`static/` 폴더 밑에 앱 폴더 만들고 파일 넣으면 끝이다. 근데 한 가지 조심할 게 있다.

GitHub Pages가 서브 경로로 배포하니까, HTML에서 asset 불러올 때 경로를 맞춰줘야 한다. `/assets/index.js`라고 쓰면 루트에서 찾으려고 해서 404 뜬다. `/pdf2md/assets/index.js`처럼 앱 폴더 경로 붙여줘야 한다.

이거 몰라서 처음에 한참 헤맸다.

지금 올라가 있는 웹앱:
- `/floor-planner/` - 아파트 평면도 그리기
- `/math/` - 수학 문제 풀이
- `/pdf2md/` - PDF를 마크다운으로 변환

## 배포는 자동

그냥 코드 push하면 GitHub Actions가 알아서 빌드하고 배포한다. 신경 쓸 거 없다.

로컬에서 미리보기 하고 싶으면 `hugo server -D` 치면 된다.

도메인은 mcu2cloud.kr 연결해뒀고 HTTPS도 자동으로 된다.

## 새 시리즈 시작할 때

바로 글 쓰지 않는다. 먼저 기획 문서 만들어서 뭘 쓸 건지 목록 정리해둔다. 안 그러면 중간에 뭘 썼는지 헷갈리고, 빠뜨리는 것도 생긴다.

`PDF2MD-시리즈-기획.md` 이런 식으로 저장해두면 나중에 찾기도 쉽다.

## 문제 생기면

**사이트가 안 보인다?** GitHub 저장소 Settings → Pages 들어가서 확인한다. Source가 "GitHub Actions"으로 되어 있어야 하고, Custom domain에 mcu2cloud.kr 들어가 있어야 한다. 저장소를 private으로 바꿨다가 다시 public으로 하면 이 설정이 날아가서 다시 해줘야 한다.

**웹앱 CSS나 JS가 안 불러와진다?** 십중팔구 경로 문제다. HTML 열어서 asset 경로에 앱 폴더명 붙어있는지 확인한다.

**Hugo 빌드가 안 된다?** `hugo --verbose` 해서 에러 메시지 본다.
